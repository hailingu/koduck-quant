# Runtime 容器和多租户测试计划

**任务**: S1-RT-002: Runtime 容器和多租户测试
**优先级**: P1 - 重要
**故事点**: 8 点
**工作量**: 2 人日
**目标测试数**: 35+ 个测试用例
**预计代码行数**: 1,200+ 行

## 1. 测试概述

本测试计划为 Task RT-002 提供完整的测试策略，验证 DuckFlow Runtime 的容器管理和多租户隔离能力。核心包括：

1. **Runtime 容器管理** - 容器的生命周期、隔离和资源管理
2. **多租户隔离** - 租户数据隔离、上下文隔离和资源隔离
3. **并发租户场景** - 并发执行、数据污染防护和竞态条件处理
4. **配额管理** - 配额的申请、释放、超限处理和恢复

## 5. 备注和注意事项

### 5.1 并发测试考虑

- 使用 `Promise.all()` 执行并发任务
- 监控资源使用情况（内存、连接数）
- 验证没有竞态条件或数据污染

### 5.2 隔离验证

- 深拷贝测试需要修改返回的对象并验证内部状态不变
- 租户隔离测试需要验证完全隔离（实体、配额、特性开关）
- 容器隔离需要验证不同容器中的管理器完全独立

### 5.3 配额管理特殊处理

- 实体配额基于实际实体计数
- 其他配额基于历史申请记录
- 无租户上下文时返回 true（不限制）
