## 5. 经验教训

### 5.1 技术层面

1. **JaCoCo BUNDLE vs CLASS 级别检查**
   - *经验*: 低覆盖率类较多的场景，BUNDLE 模式更灵活
   - *教训*: 初始使用 CLASS 级别导致阈值难以达成，切换后更合理

2. **缓存优化效果显著但需谨慎**
   - *经验*: PortfolioService 缓存优化带来 46% 性能提升
   - *教训*: 缓存一致性需要完善的 `@CacheEvict` 策略

3. **性能测试环境准备耗时**
   - *经验*: K6 + Docker 本地压测环境搭建完整
   - *教训*: 环境准备应前置，避免阻塞性能优化验证

### 5.2 流程层面

4. **任务粒度控制**
   - *经验*: P3-01~P3-09 任务粒度适中，可并行执行
   - *教训*: PMD 治理计划制定和实际执行应分开，避免"计划即完成"的错觉

5. **质量数据采集自动化**
   - *经验*: 质量指标采集脚本化，大幅提升效率
   - *教训*: 应在 Phase 早期就建立自动化采集，而非后期补充

### 5.3 协作层面

6. **文档先行策略有效**
   - *经验*: 每个任务都有详细的实施指南，执行效率高
   - *教训*: 应确保文档和代码同步更新，避免文档滞后
