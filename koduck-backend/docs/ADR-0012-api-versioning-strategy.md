# ADR-0012: 建立 API 版本管理策略（统一路径与升级流程）

- Status: Accepted
- Date: 2026-04-01
- Issue: #314

## Context

随着接口数量增长，缺少明确的版本治理策略会导致：

- 路径格式不一致；
- 不兼容变更处理方式不统一；
- 客户端升级路径不明确。

虽然当前大多数接口已使用 /api/v1/...，但仍需要制度化版本标准，避免后续演进出现分叉。

## Decision

建立统一 API 版本策略并文档化，核心决策如下：

- 路径统一为 /api/v{major}/...；
- 仅 major 版本进入 URL，minor/patch 不进入路径；
- 不兼容变更触发新 major（如 v2）；
- major 升级采用“评审 -> ADR -> 并行发布 -> 弃用下线”流程；
- 版本变更必须同步 OpenAPI、迁移说明与发布说明。

## Consequences

正向影响：

- API 路径约定统一，降低维护和认知成本；
- 版本升级路径清晰，减少客户端迁移风险；
- 评审和发布流程更可审计。

代价：

- 版本变更需要额外文档维护成本；
- 不兼容升级时需维护一段时间双版本并行。

## Alternatives Considered

1. 保持现状，仅靠口头约定执行  
   - 拒绝：执行一致性弱，风险随规模放大。

2. 使用 Header 版本而非 URL 版本  
   - 暂不采用：当前团队与调用方对 URL 版本更直观，迁移成本更低。

## Verification

- 已新增 API 版本策略文档：koduck-backend/docs/API-VERSIONING.md；
- 已在文档入口挂载版本策略链接；
- 策略与当前 /api/v1/... 路径现状一致。
