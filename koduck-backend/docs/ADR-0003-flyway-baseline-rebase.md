# ADR-0003: Flyway 迁移重置为新环境单一基线

- Status: Accepted
- Date: 2026-04-01
- Issue: #296

## Context

现有迁移目录长期演进后出现明显历史债务：

- `V1__bootstrap_schema.sql` 已包含大量历史聚合内容；
- 目录仍保留额外的 `V2..V7` 与 `V26..V27` 迁移文件；
- 编号与实际演进阶段可读性差，增加新环境初始化认知负担。

在本次决策中，项目明确要求：**不考虑旧环境兼容，直接 rebase 到新环境基线**。

## Decision

采用“单一基线 + 连续增量”的新策略：

- 迁移目录重置为 `V1__baseline.sql`（包含当前目标 schema）；
- 移除历史碎片化迁移文件（`V2..V7`、`V26..V27`）；
- 后续迁移从 `V2__...` 连续递增。

## Consequences

正向影响：

- 新环境初始化路径清晰，迁移目录结构显著简化；
- 版本语义统一，降低维护与审查成本；
- 便于后续持续增量演进。

负向影响：

- 与旧迁移历史不兼容；
- 已迁移数据库不能直接使用该重置后的迁移集进行校验/升级。

## Alternatives Considered

1. 保留旧迁移并追加文档说明  
   - 拒绝：无法真正消除编号与历史债务复杂度。

2. 兼容旧环境的双轨迁移策略  
   - 拒绝：复杂度高，且与本次“主动抛弃历史债务”的决策冲突。

## Verification

- 迁移目录已仅保留 `V1__baseline.sql`（及说明文档）；
- 旧碎片化迁移文件已移除；
- 项目可继续进行常规编译验证（`mvn -DskipTests compile`）。
