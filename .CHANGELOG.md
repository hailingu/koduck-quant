# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- Initialize repository structure and changelog file.

### Changed

- Added a standard CI quality gate workflow for both `pull_request` and `push`
  on `dev`/`main`, including:
  - Backend unit tests on Java 23
  - Data-service tests on Python 3.13
  - Dependency cache and backend report artifact upload
- Stabilized backend local/CI test runtime by aligning test profile behavior
  and using H2 as default test database for non-integration test runs.

### Fixed

- Fixed JDK 23 Mockito test compatibility by pinning test mock maker to
  `mock-maker-subclass`.
- Fixed backend test regressions caused by stale method signatures and
  endpoint test expectations.
- Improved data-service test reliability in offline/constrained environments
  by adding safe fallback data paths and compatibility handling for mocked async
  calls.
- Restored expected API behaviors used by tests:
  - `GET /api/v1/a-share/hot`
  - resilient `GET /api/v1/a-share/market/status` response under upstream failures
